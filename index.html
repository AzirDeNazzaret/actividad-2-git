Características Principales

GIT tiene varias características que lo hacen destacar:

  • Distribuido: Cada desarrollador tiene una copia completa del historial del proyecto, lo que permite trabajar sin conexión.
  • Rápido: Las operaciones en GIT se realizan localmente, lo que lo hace muy rápido.
  • Integridad: GIT asegura que los datos no se corrompan usando un hash SHA-1 para identificar cada archivo y commit.
  • Control de Versiones: GIT permite rastrear todos los cambios realizados en el código, facilitando la colaboración entre múltiples desarrolladores.

Comandos Básicos

Algunos comandos básicos que se usan en GIT son:

  • git init: Inicializa un nuevo repositorio GIT.
  • git clone [url]: Clona un repositorio existente.
  • git add [archivo]: Agrega archivos al área de preparación.
  • git commit -m "mensaje": Realiza un commit de los cambios en el repositorio.
  • git push: Sube los cambios al repositorio remoto.
  • git pull: Descarga y fusiona cambios desde el repositorio remoto.
